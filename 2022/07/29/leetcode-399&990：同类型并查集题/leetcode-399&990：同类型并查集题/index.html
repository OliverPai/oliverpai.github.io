给你一个变量对数组 equations和一个实数值数组 values 作为已知条件，其中 equations[i] = [Ai, Bi] 和 values[i] 共同表示等式 Ai / Bi = values[i] 。每个 Ai 或 Bi 是一个表示单个变量的字符串。

另有一些以数组 queries 表示的问题，其中 queries[j] = [Cj, Dj] 表示第 j 个问题，请你根据已知条件找出 Cj / Dj = ? 的结果作为答案。

注意：输入总是有效的。你可以假设除法运算中不会出现除数为 0 的情况，且不存在任何矛盾的结果。

左边的图很容易构建，而其等价于右边，如果能够得到右边的图，a/c就好算了：a→b的权值乘上b→c的权值。因此，最重要的一点在于：从左边的图转化到右边的图，而这也就是不同小图合并到一张图的过程，也是不同集合合并到一个集合的过程，因此可以考虑利用并查集完成题目。

并查集简要介绍

并查集本质是每个集合都是一棵树，树根的编号即为整个集合的编号，每个节点存储其父节点，比如p[x]表示x的父节点。并查集的基本操作为：

  1. 初始化：每个节点都作为自己的根for all x : p[x]=x

  2. 判断节点是否为树根（能判断出来，就可以知道所有子节点都属于哪个集合）：if(p[x]==x)

  3. 合并集合：比如px是x所在的集合编号，而py是y所在的集合编号，那么将两集合合并–p[px]=py

  4. 查询与路径压缩优化：当一个很深的节点，想查询其集合编号时，会走很远的路才能查到树根，这导致查询和集合合并的效率较低。路径压缩就是，当这个节点查询一遍后，它、它原来的父节点、它原来的祖父节点等等，全都将父节点指向树根，即查询一遍后，这些节点都p[x]=树根。这样再次查询时，之前的O(N)时间复杂度降为了O(1)。如下图为路径压缩的演示：

    递归实现了在查询x的集合树根节点时，x的父节点、祖父节点等，都将p[x]更新为集合的树根节点。

    实现了边查询，边做路径压缩。

而在本题中，那么还是以示例1为例，最开始无相关之间的关系，三个元素abc各自为根。之后构建了a→b这颗树，此时p[a]=b,p[b]=b，两个节点归属到同一个集合中。那么遍历到b→c时，由于目前p[b]=b,p[c]=c不在一个集合内，values的值又显示它们应当在一个集合里，这时就要把两个集合（树根为b和树根为c的集合）合并：p[b]=p[c]，至此abc三个节点进入一张图。之后再对a的集合编号进行查询时，由于路径压缩，p[a]会被更新为c。

使用并查集解决本题

前面提到，values对应的就是两个节点间，边的权重，因此除了parent数组（p[x]），记录每个节点的父节点，还需要一个weight数组，记录每个节点与其父节点之间，边的权重。由于在初始化阶段，各节点各自为根，因此，每个节点的weight[x]=1。但在合并过程中，还需要更新权重weight[x]。为了推出权重的更新公式，考虑以下几种情况：

  1. 当单纯路径压缩时：weight[x] = weight[x] * 其上面所有权重之积；

  2. 当合并x和y所在集合时：先px = find(x), py = find(y)，此时查询的过程中已经执行了路径压缩，px是x集合的树根，py也是y集合的树根，执行合并操作p[px]=py

    weight[x]仍是x到px之间的权重，目前还没有对其路径压缩，而weight[y]仍是y到py之间的权重，但是weight[px]却变了，为了计算weight[px]可以参考上图，由于values[i]刻画了x→y的比值，因此从x到py有两条路可走：x→px→py和x→y→py，所以weights[x]*weight[px] == values[i]*weight[y]，那么便可以推导出这种情况下的权重更新公式：

    weight[px] = values[i] * weight[y] / weight[x];

因此在query的时候，如果queries中某元素的两个节点都在一个集合内，比如p[m] == p[n]，那么它们之间的比值就是(m/树根)/(n/树根)=weights[m]/weights[n]；如果两节点不在一个集合内，无法计算它们之间的比值，输出-1。
