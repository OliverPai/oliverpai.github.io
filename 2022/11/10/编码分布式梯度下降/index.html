编码分布式计算：梯度下降

发表于 2022-11-10 分类于 分布式计算
论文《Improving Distributed Gradient Descent Using Reed-Solomon Codes》阅读笔记。

论文：《Improving Distributed Gradient Descent Using Reed-Solomon Codes》
出处：ISIT @ 2018
作者：Wael Halbawi(Oracle), Navid Azizan(Caltech), Fariborz Salehi(Caltech), Babak Hassibi(Caltech)
预备知识：Reed-Solomon码、梯度下降

背景

数据集规模越来越大，导致计算愈发倾向于以分布式方式执行。一个计算任务可被切分成若干子任务派发到分布式集群中的众多节点。然而此种方式面临着比较严峻的问题：

  • 慢节点拖累了整个系统的时延（慢节点也被称为stragglers）
  • 意外情况的发生可能导致某些节点无法返回结果

为了解决这一问题，编码理论应用在分布式系统中，以解决straggler效应并应对突发状况的研究，逐渐得到关注。通过在派发任务时注入冗余，能够仅通过最快的若干节点反馈整合出实际结果。

此外，AI模型的性能由训练过程显著影响，若训练充足则AI模型的性能也可能有所提升。当前训练算法主要采用梯度下降，通过计算损失函数求出梯度，更新模型参数。频繁的梯度下降依然是相当耗时的过程。

因此，此文章旨在通过分布式方式加速梯度下降，在加速的同时，能够在先验知识缺失的前提下，做到克服straggler效应和意外情况。

考虑master-worker结构，有nnn个worker（W1,W2,...,WnW_1,W_2,...,W_nW1​,W2​,...,Wn​）与一个masterMMM交互。在训练模型时，更新参数β\betaβ以最小化损失函数L(D;β)L(\mathcal D;\beta)L(D;β)，其中β∈Rp\beta\in\mathbb R^pβ∈Rp，D\mathcal DD表示给定的数据集D={(xi,yi)}i=1N\mathcal D=\{(x_i,y_i)\}_{i=1}^ND={(xi​,yi​)}i=1N​。损失函数也可以表示成另一种形式：L(D;β)=∑i=1Nl(xi,yi;β)L(\mathcal D;\beta)=\sum_{i=1}^N l(x_i,y_i;\beta)L(D;β)=∑i=1N​l(xi​,yi​;β)。因此，参数β\betaβ下完整的梯度为：

所谓分布式做梯度下降，就是将各个块的数据分给不同的worker，来并行着计算，并统一给master整合。因此假定每个worker WiW_iWi​被分配了www个数据块{Di1,...,Diw}\{\mathcal D_{i_1},...,\mathcal D_{i_w}\}{Di1​​,...,Diw​​}，对应这些数据块，此worker计算了对应每个数据块的partial gradient：g~={gi1,gi2,...,giw}\tilde g=\{g_{i_1},g_{i_2},...,g_{i_w}\}g~​={gi1​​,gi2​​,...,giw​​}。

gig_igi​是1×p1\times p1×p的行向量，g~\tilde gg~​是w×1w\times 1w×1的列向量（把gig_igi​看作元素），也是w×pw\times pw×p的矩阵（把数值看作元素）。ggg是k×1k\times 1k×1的的列向量（把gig_igi​看作元素），也是k×pk\times pk×p的矩阵（把数值看作元素）。

前面提到，不一定所有worker都是能够按时完成对应数据块的计算的，所以master要在分配的数据块中“注入冗余”（不同worker间可能有重合的Dj\mathcal D_jDj​），那么w×nw\times nw×n将不等于kkk，而是会大于kkk，在没有各worker算力之类的先验知识的情况下，最终取最快的fff个节点的结果来整合（即，系统能够容忍s:=n−fs:=n-fs:=n−f个straggler）（k>wk>wk>w）。

定理1：对于任何策略，只要kkk和www是固定值，那么可容忍的straggler数量满足s≤⌊wnk⌋−1s\le \lfloor\frac{wn}{k}\rfloor-1s≤⌊kwn​⌋−1。

此定理是其他文献证明的，而整篇文献希望优化编码策略能够让s最大，就是使其等于⌊wnk⌋−1\lfloor\frac{wn}{k}\rfloor-1⌊kwn​⌋−1。kkk和www虽然要求是固定值，但是也是可以由算法计算出来的，使得总计算时间最短的kkk和www。

编码的过程不是在master向worker派发时做编码，而是worker向master传递结果时编码。定义宏观的编码矩阵B\mathbf BB，其大小为n×kn\times kn×k，每一行Bi\mathbf B_iBi​对应一个workerWiW_iWi​，所以B\mathbf BB需要满足一个性质：

算法

算法分编码和解码两步，均不同于标准的RS编解码过程。

编码

  • 平衡矩阵

    定义矩阵M∈{0,1}n×k\mathbf M\in\{0,1\}^{n\times k}M∈{0,1}n×k满足下列条件时，为平衡矩阵（Balanced Matrix）

    1. 每一列的权重(weight)都是固定值
    2. 任意两行的权重值之差不超过1

    （权重表示，这一列或这一行的非零元素的个数）
